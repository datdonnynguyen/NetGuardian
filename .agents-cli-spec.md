# Agent Spec

## Overview

NetGuardian is an Enterprise Security Operating System for the Kaggle AI Agents capstone. The system models an Enterprise Digital Twin, ingests endpoint telemetry, creates incidents with a deterministic Incident Engine, then uses an ADK-style multi-agent layer to investigate, recommend, and verify security response actions.

## Example Use Case

Alice works in Finance and owns `Employee-01`, which has access to critical Finance assets: the `FILE-01` file server and the Payroll Database it hosts. Alice accidentally opens a malicious Excel macro. The simulated endpoint emits suspicious telemetry: PowerShell launch, DNS request, C2 network connection, and SMB scan. NetGuardian updates Enterprise State, calculates risk, creates an incident, explains the evidence, asks for human approval, executes isolation through MCP-style tools, and verifies containment.

## Tools Required

- `get_enterprise_state`: read the Enterprise Digital Twin.
- `get_incident_evidence`: read incident evidence and related business context.
- `query_threat_intel`: inspect local threat intelligence.
- `isolate_device`: isolate a device only after approval.
- `block_ip`: record a blocked IP only after approval.
- `verify_containment`: confirm containment, business impact, and rollback safety.

## Constraints & Safety Rules

- AI does not create incidents; the Incident Engine does.
- AI does not execute shell commands or directly control Docker.
- Enterprise State is the single source of truth for agent reasoning.
- High-impact actions require human approval.
- All recommendations, approvals, execution attempts, and verification results are auditable.
- No API keys or passwords are stored in code.

## Success Criteria

- A clean demo reset can reproduce the Alice malware incident.
- The Incident Engine deterministically creates a high-risk incident.
- The agent output includes evidence, reasoning, recommendation, trade-off, confidence, and required approval.
- MCP-style execution refuses unapproved isolation.
- After approval, isolation succeeds and verification confirms containment plus safety for `FILE-01` and the Payroll Database.
- The repo is runnable with Docker Compose and documented for Kaggle submission.

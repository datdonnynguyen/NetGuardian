Name: `AI Agents: Intensive Vibe Coding Capstone Project`

Host: Kaggle Community Hackathon, connected to Kaggle's 5-Day AI Agents: Intensive Vibe Coding Course with Google.

Future sessions should treat this as urgent and verify the live Kaggle page if exact timing matters.

## Core Challenge

Build an AI agent that solves a meaningful real-world problem, helps people, automates work, improves everyday living, or addresses a business/social challenge.

The project should show both:

- Product value: clear problem, users, solution, and why the agent matters.
- Technical implementation: architecture, code quality, agent behavior, tools, documentation, and demo.

## Valid Submission Requirements

A valid final submission must include:

- Kaggle Writeup.
- Media Gallery.
- Public video attached to the Media Gallery.
- Public project link.

Important constraints:

- Kaggle Writeup must be submitted before the deadline.
- Draft or unsubmitted Writeups will not be judged.
- Writeup must select a track.
- Writeup should not exceed 2,500 words.
- Video should be 5 minutes or less and published to YouTube.
- Project link should be publicly accessible and not require login or paywall.
- If live demo is not feasible, provide a public code repository such as GitHub with detailed setup instructions.
- Do not include API keys or passwords in code.

## Tracks

Available tracks:

- `Agents for Good`: projects that help solve problems for humanity, such as agriculture, public health, education, art, or literature.
- `Agents for Business`: agents solving business problems with cost or revenue impact.
- `Concierge Agents`: personal agents that simplify individual, family, or social life while keeping personal information safe and secure.
- `Freestyle`: strong agent projects that do not fit neatly into the other categories.

For `NetGuardian`, likely candidate tracks:

- `Agents for Good`, if focused on public safety, digital safety, education, or protecting people from cyber risk.
- `Agents for Business`, if focused on organizational network/security monitoring, incident triage, compliance, or cost reduction.
- `Concierge Agents`, if focused on helping individuals or families stay safe online.

The final track should be decided after the product concept is clearer.

## Evaluation

Total: 100 points.

### Category 1: Pitch - Problem, Solution, Value

30 points total.

Criteria:

- Core Concept and Value: 10 points.
- YouTube Video Submission: 10 points.
- Writeup: 10 points.

The pitch should explain:

- Problem statement.
- Why the problem matters.
- Why agents are the right approach.
- Overall architecture, preferably with images/diagrams.
- Demo of the solution.
- Build process, tools, and technologies used.

### Category 2: Implementation - Architecture, Code

70 points total.

Criteria:

- Technical Implementation: 50 points.
- Documentation: 20 points.

Judges will look for:

- Quality of architecture.
- Code quality.
- Meaningful use of agents.
- Clever and appropriate tool use.
- Useful comments explaining implementation, design, and behavior.
- README explaining problem, solution, architecture, setup instructions, and relevant diagrams/images.
